# 对于框架的猜测
## 推理过程猜测
从图中可以看出，该系统在推理阶段并不是简单地将用户输入直接送入视频生成模型，而更像是引入了一层显式的“规划—执行”中间过程。

对于用户输入的视频、图像、提示词和音频，可以推测系统首先会将这些输入送入 **PE Model**。这里的 PE Model 更像是一个 Prompt Enhancement 模块，其职责并不是生成最终结果，而是对用户原始输入进行润色、补全与规范化表达，从而输出一个更适合作为后续推理条件的 **re-prompt**。

在此基础上，系统会将两类信息共同送入 **Act VLM**：一类是用户原始的多模态输入，包括视频、图像、提示词和音频；另一类则是经 PE Model 处理后的 re-prompt。Act VLM 作为中间的决策与规划模块，可能承担两个核心职责。

首先，它会根据当前任务生成一个显式的 **执行计划（execution plan）**。这一执行计划很可能以 JSON 或结构化文本的形式存在，其中包含任务拆解、时间段规划、关键条件约束、镜头安排、参考素材使用方式等内容。

其次，它会输出一个 **模态路由决策**，即判断当前生成过程中应重点参考哪些模态信息，例如 Video、Image、Audio 或 Prompt。换言之，它不是简单地把全部输入一股脑喂给视频生成模型，而是更像一个控制器，在不同阶段决定生成模型该优先依赖哪类条件。

进一步推测，视频生成模型可能并不是一次性完成所有内容，而是按照 execution plan 分阶段执行。在执行完计划中的某一部分之后，生成结果或中间反馈会重新返回到 **Act VLM**，由其继续进行状态更新、条件修正和下一阶段决策。这意味着推理流程可能是一个轻量级的闭环过程，而不是单次前向传播。

从这个角度看，整套系统更像一个具备“规划—执行—反馈—再规划”能力的多模态视频生成代理，而不只是传统意义上的端到端视频模型。
## 训练过程猜测
训练阶段整体流程大概率与推理过程保持结构一致，只是额外引入了强化学习中的奖励评估与策略优化模块。

首先，从框架图来看，训练阶段同样包含 PE Model、Act VLM、execution plan、模态路由以及视频生成模型这几部分。因此一个自然的推测是：训练时延续了推理阶段的整条链路，只是在其末端加入了对候选视频结果的评估和策略更新。

其次，对于 **PE Model** 和 **Act VLM** 的训练方式，可能存在两种实现路径。一种可能是这两个模块已经经过预训练，在强化学习阶段主要作为相对稳定的上游模块参与推理和数据构造，而训练重心更多放在视频生成模型本身以及少量决策参数的微调上。另一种可能是这两个模块并非完全冻结，而是与视频生成模型一起进行联合训练，或者采用分阶段训练策略：先完成监督预训练，再在强化学习阶段进行部分联合优化。在这种设定下，PE Model 输出的 re-prompt、Act VLM 输出的 execution plan，以及模态路由选择本身，都可能成为强化学习优化的一部分。

最后，强化学习范式follow **critic-free** 的后训练方法，例如 GRPO 一类的 group-based policy optimization。也就是说，系统可能不依赖传统价值网络，而是通过同一输入下采样多组候选结果，利用相对比较或组间排序构建优势信号，从而完成策略更新。如果这一判断成立，那么强化学习优化的对象就不仅仅是最终视频结果本身，还包括生成结果背后的中间决策：例如 PE Model 生成的 re-prompt 是否更有利于后续生成，Act VLM 输出的 plan 是否合理，模态路由是否有效。这些中间结果如果能够被纳入 Proxy Reward 的计算，那么系统就有机会对整条“理解—规划—生成”链路进行统一优化，而不仅仅优化最后的像素生成过程。

因此，这一训练框架的重点，很可能并不只是“让视频更好看”，而是让整个系统逐渐学会生成更优的 re-prompt、更合理的执行计划以及更有效的模态调度策略。

训练过程中，大致流程可能还和推理相同，但是对于Act VLM和PE Model 来说可能是提前训练好的，也可能是和视频模型进行联合训练的。采用的RL可能还是GRPO类的Critic-Free 的范式进行RL后训练，然后对于构建的Reward Signal 来说，除去对应的video group ， Act VLM/ PE Model (如是联合训练需要,或者多阶段训练的过程)的输出的plan、路由选择和re prompt也能够输入到Proxy Reward 中，从而获取到Reward 之后，进行后续优势、pg Loss、KL等之类的RL参数。
## 对于Video Generate Model 的猜测
模型并不是单一面向视频生成的架构，而是朝着 **视频生成与视频编辑统一建模** 的方向演进。也就是说，这一模型可能不只是支持从文本生成视频，还能够统一处理多种任务形式，例如：
- 文本到视频生成
- 图像参考到视频生成
- 视频参考与视频续写
- 视频编辑或局部改写
- 音频参考驱动的视频生成
- 多模态条件联合控制下的视频生成与编辑

这一点非常关键，因为在实际应用中，用户需求往往并不是“纯生成”，而更常见的是“基于已有素材进行改写、编辑、延展或控制”。

其次，“MOE” 很可能并不是泛化意义上的概念，而是落在更具体的模型结构层面。参考当前 LLM 和视觉生成模型的发展路线，一个较为合理的推测是：**视频生成模型底层的 DiT 架构，其 FFN 部分可能被替换为 MoE 结构**。也就是说，注意力模块仍然保持主干一致，而前馈网络通过多个专家子网络进行稀疏激活，以提升模型容量、任务弹性和不同生成模式之间的适配能力。

这种设计对于统一生成与编辑场景尤其合理。因为不同任务模式对模型内部能力的需求差异很大：有些任务更强调人物一致性，有些更强调时序运动，有些更强调局部编辑边界，有些则更依赖音视频对齐。若采用 MoE 改造 FFN，模型就有可能在统一主干下通过不同专家路径实现隐式分工，从而兼顾多任务、多模态和高复杂度控制。

因此，一个相对合理的判断是：该 Video Generate Model 可能建立在统一的视频生成/编辑架构之上，并在 DiT 的 FFN 部分引入了 MoE 改造，以满足更大规模、更强可控性和更多任务形态的需求。
## Proxy Reward from Env
如果说整个系统中哪一部分最值得关注，那么 **Proxy Reward from Env** 很可能是其中最核心、也最具想象空间的部分。

原因在于，视频生成任务中的许多关键能力并不是容易通过监督学习直接学到的。无论是镜头切换的合理性、故事节奏的流畅性、音画同步的自然度，还是用户偏好的审美质量，这些都更接近“环境反馈”而非标准标签。因此，Proxy Reward 的设计在很大程度上决定了这套 RL 框架最终能学到什么。

结合 Seedance 2.0 一类视频系统的表现，可以推测其视频生成过程可能包含更频繁的镜头切换与更复杂的结构化片段组织。在这种设定下，如果背后存在一个近似“视频剪辑环境”的沙盒，那么它就可以为 Act Model 的探索提供更加真实的操作空间。考虑到字节旗下拥有剪映这类成熟的视频剪辑产品，一个相对大胆但合理的猜测是：**类似剪映的剪辑逻辑、时间轴操作或编辑环境，可能为视频生成系统中的策略学习提供了某种启发，甚至可能在内部形成了较好的训练沙盒。**

另一方面，仅依赖环境本身仍然不够。系统大概率还需要大量高质量数据训练出来的 **Reward Model**，用来提供接近 Human Preference 的代理信号。这些 Reward Signal 可能不仅评估最终生成的视频质量，还会综合考虑如下几个方面：
- 文本与视频内容的一致性
- 视频与参考图像/参考视频的对齐程度
- 音频与动作、口型、节奏之间的同步效果
- 生成过程中的镜头切换是否自然
- 编辑结果是否符合用户意图
- 整体美学质量与人类偏好分数
更进一步，如果系统确实是联合优化 PE Model 和 Act VLM，那么 re-prompt、execution plan、route 选择本身也有可能作为 Reward Model 的输入之一。这样，Proxy Reward 就不再只对“结果视频”打分，而是会对“生成这段视频的思路”一并进行评价。这一点对强化学习而言极其重要，因为它使得中间规划层真正具备被优化的可能。

## 观察到的现象
结合实际生成现象来看，这类系统虽然在整体框架上已经相当先进，但其能力边界依然比较明显，尤其是在音画对齐、复杂交互和细粒度理解推理方面。
### 音画同步的效果能力仍然是欠佳（并非完全对齐，多参考）
 当前音画同步的效果依然存在明显不足，并不能做到真正意义上的严格对齐，尤其是在多参考条件输入的情况下更容易出现偏差。
一种较为常见的现象是，Audio 更像是被切分成若干片段，然后在合适的位置插入或拼接到生成流程中，而不是作为一个连续、精确的时序驱动信号参与整个生成过程。这意味着模型可能更擅长做“粗粒度对齐”，而不是“逐帧级联动”。
另一个问题在于，音频语义如果需要与画面中的具体主体对齐，例如根据声音定位到人物再确定动作或口型内容，系统往往会出现前景与背景区分不清的问题。也就是说，模型在将 Audio 精确绑定到某个特定视觉主体时仍然存在困难，这会直接影响多角色场景、复杂镜头下的音画同步质量。
### 交互畸形能力仍然有限
在涉及人与物、人与环境、多个主体之间的复杂交互时，模型的表现仍然有限。尤其是在动作接触、物体操控、遮挡关系变化以及多主体联动的场景下，仍容易出现局部畸形、结构错误或时序不连续的问题。

这说明即便系统已经引入了高层规划与多模态控制，底层视频模型在物理交互建模、空间关系保持以及复杂动作一致性方面仍未完全突破。

### 生成理解推理能力有限
另一个较典型的问题是模型虽然具备一定的“理解生成”能力，但在更细粒度的语义区分和常识推理上依然不够稳健。
例如在参考视频的情况下，对于“脚印”这一语义，模型保持参考视频的“鞋印”，甚至直接把鞋底纹理复刻为脚印本身。这类现象说明模型在视觉生成中对于抽象词义、具身语义以及概念边界的理解仍有缺陷。它可能抓住了“印记”这一视觉形式，却没有真正区分“赤脚留下的痕迹”和“鞋底压出的纹理”之间的语义差异。
这类问题的本质，不只是生成精度问题，更是理解与推理能力尚未完全融入生成系统的问题。
